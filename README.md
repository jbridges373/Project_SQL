# Project_SQL: Parks and Recreation Database

This repository contains SQL scripts designed to create, populate, and query a relational database based on the television show *Parks and Recreation*. It serves as a beginner-friendly project demonstrating core database administration and SQL querying skills.

## 🚀 Features
* **Database Schema Creation:** Automated scripts to build tables with proper data types, primary keys, and foreign keys.
* **Sample Data Population:** Realistic mock data reflecting the characters, departments, and events from the show.
* **Data Analysis Queries:** A collection of SQL queries demonstrating data retrieval, filtering, joining, and aggregation.

## 📁 Repository Structure
* `Beginner - Parks_and_Rec_Create_db`: The core SQL script containing the database schema definition (`CREATE TABLE`) and data insertion (`INSERT INTO`) commands.

## 🛠️ Prerequisites & Installation
To run this project locally, you will need an SQL database management system (such as MySQL, PostgreSQL, or SQL Server).

1. Clone this repository:
   ```bash
   git clone https://github.com
   ```
2. Open your preferred SQL database client.
3. Open and run the `Beginner - Parks_and_Rec_Create_db` script to generate the database.

## 📊 Database Schema Overview
* **Employees:** Information on characters like Leslie Knope, Ron Swanson, etc.
* **Departments:** Details on Pawnee municipal departments (Parks & Rec, Health, etc.).
* **Salaries/Projects:** Tracking employee compensation and departmental initiatives.

## 📈 Example Queries
Once the database is created, you can practice writing queries such as:
* Finding the average salary by department.
* Joining employee details with their corresponding manager.
* Filtering employees hired within a specific date range.
